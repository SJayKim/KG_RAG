## A. PolyPharmGraph (선택) — 한국 DUR 다제약물 + 전이적 CYP 추론 〔의료〕

> 상세 설계는 [`../DESIGN.md`](../DESIGN.md) 참조.

- **한 줄**: 환자 복약 목록 전체를 받아 직접 병용금기 + **숨은 전이적 위험**(두 약이 같은 CYP 효소 경쟁→제3약 농도↑)까지 그래프로 찾고 근거 경로 출력.
- **킬러 질문**: "amiodarone·simvastatin·clarithromycin·warfarin·omeprazole 5개 중 위험 조합?" → amiodarone+clarithromycin 둘 다 CYP3A4 억제 → simvastatin 농도↑ 횡문근융해. 이 3중 조합은 어떤 문서에도 없음.
- **왜 그래프**: 조합적·기전적. `약A→억제→CYP3A4←기질←약C` 경로 + 모든 약쌍 열거를 벡터가 구조적으로 못함. RxNav DDI API 폐기(2024-01)로 단순 조회도 없음.
- **데이터**: 식약처 DUR(KOGL, 성분쌍 엣지) + openFDA 라벨(CC0, CYP 추출) + RxNav(공개, 성분명 정규화). 전부 무료, 인증 불필요.
- **노력/리스크**: M ~3주 / 낮음.
- **차별**: 한국 DUR 구조화 그래프(영어권 미사용) + 진짜 2~3홉 전이추론 + 폐기 API 공백 메우기.

## B. StareDecisis — 판례 시효성(Temporal-Validity) 에이전트 〔법률〕

- **킬러 질문**: "아직 유효 인용되는 판결 중 나중에 폐기된 선례/개정 법조문에 의존하는 것을 찾고 연쇄를 보여줘."
- **왜 그래프**: 시간축으로 유효성 전파 — 인용 다중홉 + 폐기 엣지(날짜) + 판례→법조문→개정일 교차조인. 벡터엔 '유효성' 개념 없음.
- **데이터**: CourtListener 인용맵(공공도메인, 엣지 완제품) + Eyecite(엣지 타입) + US Code USLM. 한국 확장은 국가법령정보 참조판례/참조조문. **라이선스 리스크 제로.**
- **노력/리스크**: M-L ~3~4주 / 낮음(데이터), 중(추출 정밀도).
- **차별**: 시효성은 그래프가 벡터 이기는 가장 직관적 사례. 변호사·면접관이 10초에 이해. (CourtListener가 단순 인용뷰어는 이미 제공 → 그 위 시효성 전파가 차별 레이어)

## C. MechPath — "왜 약 X가 병 Y에?" 재창출 경로 에이전트 〔바이오〕

- **킬러 질문**: "metformin과 표적/경로 공유하면서 다른 질병에 허가된 약은? 기전 경로를 보여줘" → `약→표적→경로→질병` 4+홉.
- **왜 그래프**: 재창출 후보 발굴 = 조인형 다중홉 + 집합연산. **경로 자체가 답이자 설명** — 벡터가 못 만드는 시각적 "와".
- **데이터**: PrimeKG(CC0) + Reactome(CC0, Neo4j 덤프 완제품) + Open Targets GraphQL(CC0, 라이브). BioCypher 병합. **전부 CC0, 데이터 리스크 최저, 당일 시작.**
- **노력/리스크**: M ~3주 / 최저.
- **차별**: 정적+라이브 하이브리드 + 3-way 평가(벡터 vs GNN 링크예측 vs 에이전트) — 거의 아무도 안 함.

## D. LabelVsReality — 라벨 vs 실세계 부작용 모순탐지 〔의료/약물감시〕

- **킬러 질문**: "고혈압 약 중 과거 라벨엔 없던 부작용이 현재 FAERS에 뜨는 약은? 그 신호 없는 동일계열 대체약은?"
- **왜 그래프**: ① 모순=라벨엣지(SIDER 2015) vs 실세계엣지(FAERS 라이브)의 시간축 차집합, ② 대체=동일 ATC계열 순회. 벡터는 diff/순회 불가.
- **데이터**: SIDER 4.1(수십MB, 계정X) + openFDA(CC0) + RxNav + UMLS(무료 5일). **데이터 엔지니어링 최경량.**
- **노력/리스크**: S-M ~2~3주 / 낮음.
- **차별**: 시간축 + 모순탐지 + 라벨 LLM추출 세 고급요소. 포화된 단일약 부작용 조회(1홉) 회피.

## 왜 A를 골랐나

목표가 **취업 포트폴리오 + 실제 수요**, 도메인은 의료. A는:
1. 식약처 **공식** 데이터 → 한국 채용담당자 즉시 이해 (수요 직결)
2. 성분명→RxNorm 근사매칭 → 진짜 cross-ontology 엔지니어링 (기술 깊이)
3. 폐기된 RxNav 구멍 메우기 → 툴 생태계 이해 신호
4. 전이적 CYP 추론이 1홉 조회가 아닌 **진짜 다중홉** → "그래프가 벡터를 이긴다"를 정직하게 증명 가능

데이터 전부 무료/공개, 3주 스코프. B(법률)·C(바이오)·D(약물감시)는 강력한 대안으로 보관 — 방향 바꾸고 싶으면 여기서 재선택.
